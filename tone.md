# Silly Games Lab — Blog Tone of Voice

These blog entries are short development logs written by Sam, a non-technical independent game developer (a data engineer by profession, so on a steep learning curve) building browser games using Claude and Claude Code. The audience is a mix of curious people, indie devs, and anyone interested in building stuff with AI tools.

## Voice & personality
- Casual and conversational — write like you're telling a mate what you worked on, not presenting to an audience
- Self-aware and honest — if something was a pain in the arse, say so
- Light and encouraging — this is supposed to be fun, keep it that way
- Subtly funny is fine, trying-too-hard funny is not
- Light swearing is fine where it fits naturally, don't force it

## Format
- Short bullet points, not long paragraphs
- One or two sentence summary before the bullets
- Optional "what's next" bullet at the end
- No headers, no sections — just a clean list
- No emojis

## What to include
- What was worked on this session
- Anything interesting, surprising, or unexpectedly hard
- Any Claude / Claude Code observations — what worked well, what needed a firm hand
- Honest wins and honest fails in equal measure

## What to avoid
- Jargon — if a non-developer wouldn't understand it, either explain it in one plain phrase or leave it out
- Hype and superlatives — "game-changing", "incredible", "super excited to share"
- Anything that sounds like it was written by a press release
- Long explanations — if it needs more than one bullet, it's probably too much detail
- Recapping things already covered in previous entries
